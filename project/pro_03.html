프로젝트 배경

JSP와 Spring을 이용해서 만든 스키장 사이트입니다.
세번째 JSP프로젝트이며 이전 프로젝트들과 다르게 개발에 편의를 돕는 프레임워크들을 활용해보고자 Spring과 Mybatis를 통해 개발을 했습니다.
스키장은 각각의 리조트에서 운영하는 사이트 혹은 유선전화로 예약을 하고 있기에 가격을 한곳에 모아 비교할 수 있는 사이트가 없습니다.
그점을 보완하여 스키장 가격을 비교하고 예약까지 가능한 사이트를 만들고자 했습니다. 여행지 또는 호텔, 식당 등 기존의 상용 웹사이트를 프로젝트의 예시로 적용하여 전국 예약의 스키장 정보를 보여주고 추천하며, 예약을 할 수 있는 기능을 제공하도록 웹 사이트를 제작 및 구축했습니다.
프레임워크들을 처음 쓰다 보니 익숙하지 않고 어려웠지만 Spring에 대해서 어떤 순서로 동작하고 어떻게 적용을 해야할 지 고민을 하며 Bean, Dao, Controller를 활용하여 화면을 구축하니 Spring을 이용한 CRUD 작업에 대해 많은 배움을 얻었습니다.
또 DB 연결 방식을 지금까지 해왔던 JDBC API를 이용하는 것이 아닌 Mybatis를 가지고 xml파일에 맵핑을 해보니 Connection관리 부분에서 편했고 새로운 경험을 할 수 있었습니다.
디자인을 하기 위해 사용된 CSS와 JavaScript 그리고 Bootstrap에 관해 조금 더 익숙해지는 계기가 되었습니다.

프로젝트 설명

사이트에 접속 시 다양한 기능을 가진 사이드바가 위쪽에 위치했으며 미로그인, 회원, 관리자 세 파트로 나누어 각각 다른 기능을 쓰도록 개발을 진행했습니다.
미로그인 시 공지사항만 확인을 할 수 있게 만들었고 공지사항을 제외한 다른 기능은 회원에게만 제공하도록 했습니다. 각 기능을 사용하기 위해서는 로그인을 해야하는데 회원가입을 통해 로그인을 하면 스키장 검색 및 예약, 회원 정보 수정, Q&A 게시판 이용 등 다양한 기능을 사용할 수 있습니다.
또 관리자는 회원이 할 수 없는 회원 삭제, 리조트 등록 및 관리, 공지사항 삭제 등 관리자만이 가능한 기능을 사용할 수 있도록 차이점을 둬 사이트를 구축했습니다.
기존과 다르게 공지사항과 Q&A게시판에 검색 필터를 추가해 키워드만으로도 관련 글이 뜨고 확인이 가능합니다.

문제 정의

JSP페이지에서 JAVA를 포함한 여러 코드들이 뒤섞이니 코드 자체가 복잡해져서 가독성이 떨어지는 현상이 나타났습니다.
공지사항 및 Q&A 등과 같이 페이징 처리를 해야되는 곳에서 동작이나(예상치 못한 페이지 범위를 요청받았을 때 등) 페이지 계산같은 것들이 원하는 대로 작동이 되지 않았습니다.
또 데이터들을 얻으려면 최소 2번의 API 요청(데이터 요청, 데이터 카운트 콜)을 통해 데이터를 가져와야 하는데 한번의 요청으로 모든 데이터를 가져오게 되면 데이터가 매우 많을 경우에
성능 이슈가 생길 수 있습니다.
DB연동을 할 때 JDBC 드라이버를 로드해서 연결해서 구축하는 방식은 코드가 복잡할 뿐더러 변경이 있을 시 수정하거나 컴파일을 해야되고 데이터 접근 속도나 생산성 면에서 떨어지는
문제를 알게되었습니다.
로그인을 하지 않아도 Q&A 게시물을 확인할 수 있는 문제점이 발견되었습니다.

문제 해결

el표현식과 JSTL을 통해 가독성을 상승 시키고 문법체계가 직관적이게 바꿔줬습니다.
여러 시행착오를 겪어 페이지네이션 처리용 클래스를 적재적소에 맞게 사용했습니다.
Spring Data JPA 라이브러리의 Page와 Pageable을 이용하여 페이지 크기와 페이지 순서만 전달하면 DB에서 해당 페이지에 해당하는 데이터만 가져와서 해결이 가능한데 ORM방식이 아닌 SQL Mapping 방식인 Mybatis를 활용하는 바람에 쓰지는 못했습니다.
하지만 Mybatis를 사용하여 데이터 접근 속도를 높혀주는 Join 맵핑과 쿼리문을 직접 작성하므로 최적화된 쿼리를 구현 가능했습니다.
세션 정보를 확인하고 미로그인 시 redirect를 통해 요청한 URL이 아닌 로그인 페이지 URL로 이동할 수 있게 조치를 취했습니다.

프로젝트 기능
  • 관리자 : 예약 내역 확인, 리조트 관리, 이용권 관리, 회원 목록 보기.
  • 사용자 : 검색, 예약 내역, 회원 정보 수정, 회원 정보 보기, 회원 탈퇴.
  • 공통 : 로그인 및 로그아웃, 문의글(Q&A)관련 기능, 공지사항, 정보 수정, 상세 보기.
맡은 역할

- 관리자의 예약 내역 확인, 리조트 관리, 이용권 관리, 회원 목록 보기 화면 및 기능 구현

- DB 연동 작업

- 전체적인 디자인 작업

사용된 기술 및 언어
  • IDE : Eclipse, Oracle SQL Developer
  • 언어 : JAVA (JDK 1.8), HTML/CSS, JavaScript
  • 프레임워크 : Spring, Mybatis, Bootstrap
  • 서버 : JSP, Apache Tomcat 9.0
  • DB : Oracle DB
  • 라이브러리 : jQuery, JSTL

프로젝트 시연

로그인과 로그아웃, 회원 정보 수정 및 상세보기 페이지와 리조트 상세보기, 수정, 삭제, 이용권 추가 페이지입니다.

공지사항 및 Q&A 필터 검색 기능이 담긴 페이지와 관리자 전용 리조트 목록 조회 및 작성 페이지입니다.

관리자 전용 회원 정보 수정 및 상세보기 페이지와 사용자만 가능한 리조트 예약관련 페이지입니다.

공지 사항 조회페이지와 (일반사용자 이므로 작성,수정,삭제 권한이 없음) Q&A 문의글 작성 및 수정 페이지입니다.

<c:set var="whologin" value="0" />
    <c:if test="${empty sessionScope.loginfo}">
        로그인 하지 않은 경우
        <c:set var="whologin" value="0" />
</c:if>
<c:if test="${not empty sessionScope.loginfo}">
    <c:if test="${sessionScope.loginfo.member_id == 'admin'}">
        관리자로 로그인한 경우
        <c:set var="whologin" value="2" />
    </c:if>
    <c:if test="${sessionScope.loginfo.member_id != 'admin'}">
        일반 사용자로 로그인한 경우 
        <c:set var="whologin" value="1" />
    </c:if>
</c:if>
