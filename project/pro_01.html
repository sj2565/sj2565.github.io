메모장 및 게시판 홈페이지

프로젝트 배경

처음 JSP를 배운 후 JAVA와 함께 웹 페이지로 구현한 사용자 메모장 및 게시판 홈페이지입니다.
프로젝트라고 보기에는 미흡하지만 이 프로젝트를 통해 백엔드에 관심이 생겼고 어떤식으로 서버를 구축하며 운영하는지 기초를 다져서 더 큰 프로젝트를 개발하고싶었습니다.
JSP가 어떤 구조로 만들어지고 활용되는지 연습해 보기 위해 개인으로 진행했습니다.
간단하면서도 주요 핵심이 들어간 기능을 활용해 보고자 웹 사이트라면 안 쓰일 수가 없는 로그인 기능과 회원가입, DB와의 연동이 들어갔으며 로그인 성공 시 나타나는 메인 홈페이지 화면을 통해 글을 작성할 수 있게 개발했습니다.
프로젝트를 진행하면서 객체를 가져오기 위해 사용되는 자바빈 클래스의 Getter, Setter 및 JSP에서 자바빈을 명시하기 위한 여러 태그 사용 방법과 JSP의 전용 태그와 문법을 알 수 있었고
JSP에서 DB와의 연동 방법도 익힐 수 있었습니다.

프로젝트 설명

메모장 홈페이지에 접속하기 전에 로그인을 해야만 볼 수 있도록 설정을 하고 아이다가 없다면 회원가입을 통해 아이디와 비밀번호를 생성할 수 있습니다.
생성된 아이디와 비밀번호를 입력하면 홈페이지에 접속이 가능하고 DB와 연동을 해서 회원 정보를 저장합니다.
접속한 화면에는 INSERT, DELETE, UPDATE기능이 있는 메모판이 나오며 사용자가 쓰고 싶은 기능을 사용하면 됩니다.

문제 정의

간단한 메모장 홈페이지라 순조롭게 진행될 줄 알았지만 회원가입 부분이 구현이 잘 안되었던 건지 회원가입 진행이 되지 않았던 문제가 있었습니다.
DB와의 연동도 문제없이 잘 되었고 테이블 양식에 따라서 작성한 SQL문법에도 에러가 있는지 확인했지만 없었습니다.
이 외에도 자바빈 액션 태그 및 예외 처리 등을 확인해 보았지만 틀리게 썼다거나 오류는 없었습니다.
예외처리 없이 단순하게 onClick 이벤트를 통해 링크를 설정하는 바람에 로그인을 안하고 URL수정만 해도 메모장 페이지에 접근할 수 있었던 문제가 발생했습니다.

문제 해결

JAVA에서 DB로 쿼리문을 전송할 때 사용할 수 있는 인터페이스 중 하나인 PreparedStatement를 사용해서 데이터를 전달하려고 했습니다.
PreparedStatement는 Statement와 다르게 각각 인수에 대해 위치홀더(placeholder)를 사용하여 SQL 문장을 정의할 수 있지만 위치 홀더에 Column 값을 쓰는 것이 아닌 물음표(?)로 표기해야 하는 것을 모르고 직접 값을 써줘서 오류가 났었습니다. 위치 홀더에 물음표(?)를 써주니 오류가 사라지고 프로그램 동작이 성공적이었습니다.
이번 오류로 PreparedStatement는 Statement랑 다르게 컴파일이 미리 되어있고 더 좋은 성능을 가진 것과 또 특수문자를 자동으로 파싱해 주기 때문에 SQL injection 같은 공격을 막을 수 있어서 보안 쪽으로도 강화되어 있었음을 알 수 있었습니다.

프로젝트 기능
  • 로그인 및 회원가입 기능.
  • DB와의 연동을 통한 데이터 확인 기능.
  • 데이터 삽입, 삭제, 수정 기능.
사용된 기술 및 언어
  • IDE : Eclipse
  • 언어 : JAVA (JDK 1.8), HTML/CSS, JavaScript
  • 서버 : JSP, Apache Tomcat 9.0
  • DB : MySQL
프로젝트 시연

로그인 페이지와 회원가입 페이지입니다.
